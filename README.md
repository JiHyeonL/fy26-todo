## 요구사항
- [ ] 사용자는 ToDo를 생성/조회/수정/삭제할 수 있어야 함
- [ ] ToDo를 생성한 사용자를 구분할 수 있어야 함
- [ ] 태그로 ToDo의 카테고리를 분류할 수 있어야 함
- [ ] 생성일, 마감일, D-Day 정보를 포함해야 함
- [ ] ToDo 항목의 순서를 변경할 수 있어야 함
- [ ] 사용자, 태그, 완료 여부를 조건으로 ToDo를 필터링할 수 있어야 함
- [ ] ToDo를 다른 사용자에게 조회만 가능한 상태로 공유할 수 있어야 함
- [ ] ToDo를 생성한 사용자만 수정/삭제할 수 있어야 함

## 고려사항
### 1. 현재 관리자(admin)의 ToDo 수정/삭제 권한  정책이 결정되지 않은 상황
추후 다음 중 하나로 결정될 예정
1. **관리자는 모든 사용자의 ToDo를 수정/삭제 가능**
2. **관리자도 본인이 생성한 ToDo만 수정/삭제 가능**
### 2. 차후에 삭제한 ToDo를 되돌리는 요구사항이 추가될 수 있음
